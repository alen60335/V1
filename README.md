# 環陣行者（Ring Walker）— 陣法類銀河城 V1

依《陣法系統設計文件.md》實作的 HTML5 橫向捲軸類銀河戰士惡魔城原型。
純前端、零依賴，直接用瀏覽器開啟 `index.html` 即可遊玩（建議 Chrome / Edge）。

## 執行方式

- 直接雙擊 `index.html`，或
- 起本機伺服器：`python -m http.server 8873 --directory .` 後開 `http://localhost:8873/`

## 操作

控制採「空洞騎士式」左手動作叢集：右手方向鍵移動，左手 A S D F 打元素、X 跳、C 攻擊、Space 起陣。破陣是**模式切換**——按一次 Shift 進入破陣模式，元素改輸入到破陣序列，Space 發動後（不論成敗）自動退出。

| 動作 | 按鍵 |
|---|---|
| 移動 | ← → |
| 跳躍 | X（或 ↑）；習得風翼玉後空中可再按一次＝**二段跳** |
| 蹬牆跳 | 習得岩爪玉後：空中朝牆推＝貼壁滑落，此時按跳躍蹬牆而上 |
| 法杖揮擊（弱近戰） | C |
| 輸入施放元素 | A火 S水 D土 F風 |
| 起陣（施放） | Space |
| 進入／退出破陣模式 | Shift |
| 輸入破陣元素 | 破陣模式下 A/S/D/F |
| 發動破陣 | 破陣模式下 Space |
| 撤銷上一個輸入 | Backspace |
| 刻紋／交易選單 | Q（站在點燃的祭壇旁＝刻紋裝備；站在遊商旁＝購物） |
| 陣式表 | Tab |
| 洞窟輿圖 | M（房間式地圖：走訪過的房間才顯現；方向鍵平移） |
| 暫停／靜音 | P／N |

- **輸入預覽**：輸入元素時左下即時顯示成品名稱與狀態（如「→ 追蹤火球（未成圓）」）。
- **起始標記**：所有陣圈（自己、敵陣、結界）的第一個元素放大並帶金環，陣心金色指針恆指起始，旋轉中也讀得出序列從哪開始、往哪讀。
- **起陣**後不論成功或失敗，都會清空目前輸入的元素。
- **破陣**確認後（成功或失敗）都會清空破陣序列並跳回一般模式；破陣模式時 HUD 右下會亮起粉紅「● 破陣模式」。

> 綁定集中在 `js/game.js`：元素在 `KEY_ELEM`、跳/攻擊/確認在 keydown 的 `KeyX`/`KeyC`/`Space` 判斷、破陣模式切換在 `toggleBreakMode`、移動在 `updatePlayer` 的 `ArrowLeft/ArrowRight`。

## 流程（約 40～60 分鐘，三區域三頭目）

**區域一・起始洞窟群**：教學告示 → 結界一（土火）→ 裂谷（風系位移）→ 祭壇＋陣槽玉 →
咒師走廊（對敵破陣、高台陣槽玉）→ 深層結界（五元素）→ **頭目一「深陣咒主」**（自癒與反噬大陣）。

**區域二・幽泉水道**（擊殺頭目一後封印崩解）：入口即見**風翼玉（二段跳）**→ 晶蝠（飛行）→
命玉暗室與上層迴廊（爆晶蟲看守的秘寶）→ 晶窟跳台（盾岩獸、水咒師；**陣槽玉三高台需二段跳**）→
**反噬結界**（破除必受反傷）→ 棘晶弩與反噬咒師 → **頭目二「疾風梟主」**（飛行頭目）。

另有秘密：裂谷底部東側岩隙藏有暗道（谷中石碑有暗示）。

**區域三・祖陣迴廊**：休息室（祭壇、靈力玉、命玉、**岩爪玉＝蹬牆跳**）→ 精英走廊與**蹬牆井**
（爬上頂部密室拿命玉四）→ 六元素結界（需三顆陣槽玉）→
**最終頭目「祖陣墟主」**（六元素大爆裂、帶反噬的自癒陣、召喚晶蝠）→ 歸環之門通關。

能力進程（空洞騎士式）：**風翼玉**＝擊敗頭目一後取得，解鎖二段跳與晶窟高台；
**岩爪玉**＝擊敗頭目二後取得，解鎖蹬牆跳與蹬牆井。HUD 左上顯示已習得能力徽章。

## 刻紋系統（流派配裝）

空洞騎士護符式：**刻紋槽 3 格**（遊商可擴到 4），每枚刻紋佔 1～2 格，
**只能在點燃的祭壇（Q）換裝**。擊殺敵人掉**靈砂**（◈），遊商在祭壇室。

| 刻紋 | 槽 | 效果 | 取得 |
|---|---|---|---|
| 烈焰之核 | 2 | 火系傷害 +35% | 遊商 ◈40 |
| 泉心佩 | 1 | 水系治療 +40% | 遊商 ◈30 |
| 鐵壁符 | 2 | 受傷 -25% | 遊商 ◈60 |
| 刻紋擴環 | — | 刻紋槽 +1（上限 4） | 遊商 ◈80 |
| 迅陣之絡 | 2 | 施展窗口 -30% | 裂谷底暗道 |
| 追獵之目 | 1 | 法杖 9→14、揮速+、命中回靈 4 | 水道上層迴廊 |
| 蝕魂之鎖 | 1 | 擊殺回復 8 生命 | 走廊C（精英看守） |
| 破軍之爪 | 2 | 破陣傷害 10→35、暈眩 3.5 秒 | 蹬牆井頂密室 |
| 輕靈羽 | 1 | 位移陣距離 +40%、移速 +10% | 梟主巢穴 |

流派範例：**烈焰爆發**（烈焰之核＋迅陣之絡）、**破陣獵手**（破軍之爪＋迅陣之絡）、
**法杖武僧**（追獵之目＋蝕魂之鎖＋輕靈羽）、**不動堡壘**（鐵壁符＋泉心佩＋蝕魂之鎖，需擴環）。

## 音樂與環境音

BGM 為 **WebAudio 程序化生成**（零外部音檔）：低音鋪底（雙鋸齒微失諧＋低通）、
調式內隨機旋律音（洞窟回聲延遲線）、風噪與水滴環境聲。四種氛圍自動切換：

| 氛圍 | 觸發 | 特色 |
|---|---|---|
| cave | 區域一 | A 小調五聲、疏落 |
| water | 區域二 | C 調、水滴密集 |
| ruin | 區域三 | F♯ 陰暗音程、低沉 |
| battle | 任一頭目交戰中 | 低音脈動、旋律加快 |

實作在 `game.js` 的 `Music` 模組；N 鍵靜音同時作用於音效與音樂。

## 規則推導的重要發現（回饋給設計）

克制＋閉合規則會自然導出：**合法的純二元素陣只有 4 種**（火土、土火、水風、風水）。
另外 8 種組合中「主元素克制修飾元素」者（火風、水火、土水、風土）無法以兩元素成圓，
必須加入第三個元素才合法——這不是實作限制，而是規則的必然結果，反而讓組合有了
自然的成本梯度（強力組合需要更多陣槽）。V1 採用的建議輸入：

（下列以按鍵表示，A火 S水 D土 F風）

- 2 槽：火種 `A D`、爆破 `D A`、流療 `S F`、流體位移 `F S`
- 3 槽：延燒 `A D S`（土為間接）、追蹤 `A F D`、爆療 `S A F`、泉湧 `S A D`（火為間接）、
  回復陷阱 `D S A`、感應陷阱 `D S F`（水為間接）、衝刺 `F D A`（土為間接）、錨點 `F D S`
- 5 槽：反噬陣例 `A F S F D`（兩個風皆為間接 → 反噬）

## V1 的取捨（文件未定或本作簡化）

- 敵人不會反過來破玩家的陣（對稱破陣留待後續版本）。
- 玩家自組反噬陣目前只有視覺標記（因敵人不破陣，副效果不會觸發）。
- 破陣成功附帶：中斷施法、目標暈眩 2.5 秒、小額反傷 10（遊戲性選擇，非文件規定）。
- 「空間」元素、8 格陣槽上限成長曲線尚未實作（V1 上限 5）。

## 美術素材說明

大圖素材由**本機 Stable Diffusion（Forge / AOM3A3 模型）**生成，放在 `assets/`：

| 檔案 | 用途 |
|---|---|
| `bg_cave.png` | 遊戲洞窟背景（水平平鋪＋視差，繪製時壓暗） |
| `title_scene.png` | 標題畫面大圖 |
| `boss_portrait.png` | 頭目一 深陣咒主 登場卡＋血條頭像 |
| `boss2_portrait.png` | 頭目二 疾風梟主 登場卡＋血條頭像 |
| `boss3_portrait.png` | 最終頭目 祖陣墟主 登場卡＋血條頭像 |
| `player_portrait.png` | HUD 左上玩家頭像 |

載入集中在 `js/assets.js`；`game.js` 各繪製函式以 `Assets.has(key)` 判斷——
**圖片載入成功就用圖，失敗自動 fallback 回程式化繪製**，所以刪掉 `assets/` 遊戲仍能跑。

會動的小角色（玩家／小怪／咒師）仍為程式化繪製：AI 難以產生一致且乾淨的小尺寸
透明動畫幀，程式繪製在此更穩定。之後若要換成 sprite，改 `drawPlayer`／`drawEnemy` 即可。

### 重新生成素材（本機 Forge）

1. 以 `--api` 啟動 Forge（該專案的 `run_api.bat` 已設好，API 在 `127.0.0.1:7860`）。
2. `POST /sdapi/v1/txt2img`，`override_settings.sd_model_checkpoint = "AOM3A3_orangemixs"`，
   base64 結果存成上表對應檔名即可（prompt 見 git 歷史的接素材 commit）。

## 除錯掛鉤

隱藏分頁時 rAF 不會觸發；自動化測試可用 `window.RingWalker.step(ms)` 手動推進影格、
`RingWalker.debug` 讀取狀態、`RingWalker.resume()` 恢復正常迴圈。
